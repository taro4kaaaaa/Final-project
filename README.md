# 📌 Frontend Developer Portfolio

Интерактивное портфолио-приложение, демонстрирующее мои навыки во frontend-разработке.

Основной фокус — создание чистой структуры проекта, работа с TypeScript, переиспользуемые компоненты и пользовательский интерфейс.

## 🧠 Архитектура проекта

Проект организован по компонентному подходу с разделением ответственности:

- `components/` — UI-компоненты (About, Contact, Projects, ThemeButton и др.)
- `context/` — глобальное состояние (например, тема приложения)
- `i18n/` — конфигурация локализации
- `api/` — подготовка к работе с внешними данными
- `assets/` — статические ресурсы

Подход позволяет:
- изолировать логику
- переиспользовать компоненты
- упростить поддержку

---

## ⚡ Основной функционал

- отображение информации обо мне и проектах
- переключение темы (light/dark) через Context API
- поддержка локализации (i18n)
- адаптивный интерфейс
- компонентная структура приложения

---

## 💣 Ключевые решения

- Использование Context API для глобального состояния (тема)
- Кастомный хук `useTheme` для безопасной работы с контекстом
- Разделение компонентов по зонам ответственности
- Использование TypeScript для типобезопасности
